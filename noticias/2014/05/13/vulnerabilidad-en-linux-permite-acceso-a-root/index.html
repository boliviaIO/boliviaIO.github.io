Vulnerabilidad en Linux permite acceso a root

La vulnerabilidad podría permitir a un usuario local elevarse para tener privilegios de root, la misma está presente desde el kernel 2.6.31-rc3 al 3.15-rc5.

CVE-2014-0196

El problema es ocasionado por una función llamada n_tty_write presente en (drivers/tty/n_tty.c), el cual maneja el acceso correcto a la terminal virtual usando las banderas(flags en ingles) «LECHO & !OPOST», el cual permite hacer daños en el área de la memoria del kernel. Uno de los cuatro casos encontrados, usando un exploit es posible bloquear el kernel, el exploit solo funciona en versiones del kernel >= v3.14-rc1, por que a partir de esta fue adicionada:

tty: Halve flip buffer GFP_ATOMIC memory consumption

git.kernel.org/cgit/linux/kernel/git/torvalds/linux.git/commit/?id=acc0f67f307f52f7aec1cffdc40a786c15dd21d9

Del cual depende el exploit.
